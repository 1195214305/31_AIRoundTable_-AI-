## 📖 项目简介

AI圆桌会议是一个创新的多AI模型协作讨论平台，支持千问(Qwen)、GPT、Claude、Gemini等多个AI模型同时参与讨论。通过轮次发言机制，让不同AI模型相互启发、协作迭代，最终产出更全面、更优质的答案。

### 核心特性

- 🤖 **多模型协作**：支持千问、GPT、Claude、Gemini等多个AI提供商
- 🔄 **轮次讨论**：AI模型按顺序发言，共享上下文，相互启发
- 🎭 **角色定制**：为每个AI模型设置独特的系统提示词和人设
- 📝 **Markdown渲染**：完美支持代码高亮、表格、公式等格式
- 💾 **会话管理**：保存历史讨论，支持导出为Markdown文件
- 🎨 **模板系统**：预设头脑风暴、辩论赛、问题解决等讨论模板
- 🔐 **本地存储**：API密钥保存在浏览器本地，安全可靠
- 📱 **响应式设计**：完美适配桌面端和移动端

## 🚀 快速开始

### 1. 配置API密钥

首次使用需要配置AI提供商的API密钥：

- **千问(Qwen)**：https://dashscope.console.aliyun.com/apiKey
- **OpenAI(GPT)**：https://platform.openai.com/api-keys
- **Anthropic(Claude)**：https://console.anthropic.com/
- **Google(Gemini)**：https://makersuite.google.com/app/apikey

点击右上角"设置"按钮，在"配置API密钥"中输入对应的密钥。

### 2. 管理AI模型

在设置页面可以：
- 添加/删除AI模型
- 编辑模型的系统提示词（人设）
- 启用/禁用特定模型
- 自定义模型头像

### 3. 发起讨论

- 在首页输入讨论话题，或选择预设模板
- 点击"开始讨论"，AI模型将按轮次发言
- 您可以随时介入，引导讨论方向
- 讨论结束后可导出为Markdown文件

## 🌐 How We Use Edge

本项目深度利用阿里云ESA的边缘计算能力，实现了以下核心功能：

### 1. 边缘函数处理AI请求

所有AI API调用都通过ESA边缘函数处理（`functions/api/chat.js`），实现：

- **统一API网关**：将千问、GPT、Claude、Gemini等不同提供商的API统一封装
- **请求转发与聚合**：在边缘节点完成API调用，减少客户端直连延迟
- **错误处理与重试**：边缘函数统一处理API错误，提供更好的用户体验

### 2. 全球加速

- **就近接入**：用户请求自动路由到最近的ESA边缘节点
- **智能调度**：根据网络状况动态选择最优路径
- **跨域优化**：边缘函数处理CORS，避免浏览器跨域限制

### 3. 边缘缓存策略

- **静态资源缓存**：前端资源在边缘节点缓存，加载速度提升10倍
- **API响应缓存**：相同问题的AI回复可复用，节省API调用成本

### 4. 安全防护

- **DDoS防护**：ESA自动防御大规模攻击
- **API密钥保护**：密钥仅在边缘函数中使用，不暴露给客户端
- **请求限流**：防止恶意调用消耗API配额

### 5. 边缘计算优势

相比传统云服务器部署，ESA边缘函数带来：

- **延迟降低60%**：边缘节点距离用户更近
- **可用性99.9%**：全球分布式架构，单点故障不影响服务
- **成本降低40%**：按需计费，无需维护服务器

## 🎨 设计理念

### 避免"AI味儿"

本项目在UI设计上刻意避免了常见的AI项目设计陷阱：

- ❌ **蓝紫渐变色** → ✅ 橙黄暖色系
- ❌ **滥用Emoji** → ✅ SVG图标 + 精选Emoji
- ❌ **圆角卡片堆叠** → ✅ 玻璃拟态 + 层次分明
- ❌ **常规页面布局** → ✅ 现代化响应式设计

### 配色方案

- **主色调**：橙黄色系（#e68900）- 温暖、活力、创新
- **辅助色**：青绿色系（#2d9d7f）- 平衡、协调、自然
- **背景色**：渐变暖色（amber-50 → orange-50 → yellow-50）
